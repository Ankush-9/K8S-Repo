# HELM
=========================
~ We have package managers as
	Amazon Linux/Red hat : yum
	Ubuntu : apt
~ Likewise, HELM is also a package manager to install the required softwares in k8s cluster
~ It is basically an easier approach to install required softwares similar to yum/apt package managers in Linux Distributions

~ HELM uses charts in order to achieve this
~ Charts refers to collection of configuration files i.e. all the manifest-yml files
~ Using HELM charts we can install Prometheus server ,Grafana Server,ArgoCD and so on.....

~ Sometimes we require to install some of the softwares in our cluster which require lot of yml files
~ We can use the concept of package managers to install these types of softwares I.e. Helm

~ Softwares built for kuberenetes include metrics server, grafana and Prometeus, ArgoCD and so on…
~ We have two options to install these types of tools :
	—> creation of manifest files
	—> Helm

~ Using Helm the approach to install the softwares becomes easier, as Helm already has these manifest files

~ Helm helps you manage Kubernetes applications — Helm Charts help you define, install, and upgrade even the most complex Kubernetes application.
—> Official Helm website : https://helm.sh
—> ArtifactHub : https://artifacthub.io
	Artifact Hub is a CNCF incubating project
—> The ArtifactHub is the place where we can search for softwares and we get links to install those softwares. —> It is analogous to DockerHub

~ Helm Repo : It has all the helm repositories which is publicly available

~ First we need to install Helm to use it
~ The following command is used to install Helm
	curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
~ To verify installation hit the following command
	helm version
