# ArgoCD:
~ CI/CD ~ Continuous Integration and Continuous Deployment
~ Pipeline ~ A pipeline means it has continuous tasks to be carried out in one go
~ Deploying application ~ Giving life to the code

~ For an existing deployed application, if there are any new commits in the app code in Github or so ,then whatever image is deployed in the end user requires repackaging, testing, code-reviews(using SonarQube) and so to create a new image and deploy the new image for every update
~ We can automate this process/jobs/tasks i.e. the moment developers make any changes to the code ,immediately the above process requires to be followed —> We can automate the process using Gitlab, GitHub actions and Jenkins,
This is only called CI/CD i.e. automation to achieve continuous deployment

~ ArgoCD is used to perform the deployment in K8S cluster while the pipeline concept is used in Jenkins

~ Basically ArgoCD is a continuous deployment tool for Kubernetes cluster where, Argo is the tool and CD refers to Continuous Deployment
—> If changes are made in manifest yml files these changes need to be reflected
Eg. replicas are changed from 4 to 10 ,to do so we have make changes in k8s-manifest files and to do so we have to make changes and commit those changes in GitHub repository
       Using ArgoCD the changes will be directly reflected to the k8s cluster I.e. automatic deployment of application changes

~ ArgoCD will continuously monitor the Git repository for the changes in the repository and automatically apply them to the K8S cluster

# Installation of ArgoCD:
~ Go to artifacthub
~ Search for ArgoCD
~ Always select the verified publisher and install from their links
~ Select a publisher and click on “install” and follow the command instructions
        -> Add the repo first
            helm repo add argo https://argoproj.github.io/argo-helm
        -> Update the repo
            helm repo update
~ kubectl create ns ankush-ns
*Whenever we install it is recommended not to directly install in the default namespace —>      Create a new namespace and install there
~ helm install argocd argo/argo-cd —namespace ankush-ns
*We get default username as admin and to get the default password we can use the following the command
        kubectl -n ankush-ns get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
*This all the above is done to access the server UI

# Observations:
~ kubectl get all -n ankush-ns
        We get all the manifest-yml which are used behind the scene in the namespace where we have installed argocd
        The above is used to illustrate the uses of Helm charts, about how easier it is to install softwares using Helmcharts

~ The argocd server uses the “ClusterIP” Service type but to externally expose the app of the k8s cluster we may use LoadBalancer/NodePort Service
~ We have to make changes here

~ Use the command below to expose ArgoCD server
        kubectl patch svc argocd-server -n ankush-ns -p '{"spec": {"type": "LoadBalancer"}}'

~ Now copy the LoadBalancer URL to access the Server UI on the web browser
~ Provide the username “admin” and password using the password access command and sign in
~ We’ll be logged in to the ArgoCD web server UI
~ For now we don’t have any deployed applications here
