#Grafana and Prometheus :
~ Grafana and Prometheus are used to monitor the entire cluster
~ We can also use these tools for monitoring servers

# Observability:
~ In the world of devops observability refers to the tools and methodologies that helps developments and DevOps team to log,collect and analyse performance to get insights
->Monitor 
->Logs 
->Alerts

#Grafana:
~ Grafana is built on the principle that data should be accessible to everyone in our organisation 
~ Grafana helps to facilitate a culture where data can be easily used ;helping to break-down and empower teams for decision making
~ Grafana provides us with a dashboard where we can visualise the data from our K8S Cluster or different cloud services or even from google sheets

*Grafana is basically used for Visualisation

~ Grafana internally uses a datasource for visualisation named as Prometheus 

#Prometheus
~ Prometheus is an open source tool for monitoring which acts as a datasource for Grafana
~ Prometheus is the tool which actually monitors the applications or servers —> It collects the metrics(information about parameters like cpu, memory) over time while,
   Grafana creates a dashboard and displays the information

*We do have another datasource tool named as Grafana Loki which can be used to monitor

*We can call these tools as log aggregation tools —> One which collects logs

#Installation of Metrics server:
~ We are required to install the metrics server
~ We can use Ai tools to get commands to install merics server
~ Also, there might be scenarios such as error in links,resolving some errors or so at any instance of time
~ For the time of installation of mertics server the used command is as follows :
   kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml
   kubectl patch deployment metrics-server -n kube-system --type='json' -p='[{"op":"add","path":"/spec/template/spec/containers/0/args/-","value":"--kubelet-insecure-tls"}]'

*For performing everything creation of namespace is recommended -> Not compulsory but highly recommended

~ kubectl create ns prometheus
~ kubectl create ns grafana

#Installation of Prometheus:
~ Go to artifacthub and search for “promotheus”
~ Select the one from a verified publisher
~ Click on “install” 
~ Copy and perform the command based operations:
	Add Repo
	-> helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
	Install Prometheus
	- > helm install my-prometheus prometheus-community/prometheus --version 28.13.0 —namespace prometheus
~ We get a message as :
	For more information on running Prometheus, visit:
	https://prometheus.io/
~ The above message means prometheus is running in this url we can access prometheus by visiting this URL
~ Update the helm
	helm repo update
~ kubectl get all -n prometheus

#Installation of Grafana:
~ Go to artifacthub.io and search for “grafana”
~ Select the one from a verified publisher
~ Click on “Install”
~ Copy and perform the command based operations:
	Add Repo
	-> helm repo add grafana-community https://grafana-community.github.io/helm-charts/
	Install Grafana
	-> helm install my-grafana grafana-community/grafana --version 11.3.0 —namespace grafana
~ Update the helm
	helm repo update
~ kubectl get all -n grafana

*Now verify whether the two repositories are installed or not by using the following command~
~ helm repo list
*The above command lists all the repositories which are actively present

~ Change Service Type of Grafana as ~
	 kubectl patch svc my-grafana -n grafana -p '{"spec": {"type":"LoadBalancer"}}'

~ kubectl get all -n grafana
   Now using the external IP of my-grafana i.e Load Balancer URL we can access the Server UI for Grafana
~ We can check the pod status in grafana namespace using the command below~
	kubectl get pods -n grafana

#For Deletion of resources use the following commands~
~ kubectl delete all —all -n grafana
~ kubectl delete all —all -n prometheus

*If we are working as a fresher, most likely we are part of the L1 team where we monitor dashboards 
*We use Grafana ,where Grafana uses Prometheus
*Prometheus collects metrics over-time

*We can create a common namespace for both prometheus and grafana
Eg. kubectl create ns monitoring

#Practical Implementation:Setting 
~ As we have created a cluster again the next day our namespaces are gone(deleted), while repos are present already
~ Follow the instructions as below
~ Get connected to VM where everything is configured and create the cluster
~ helm version
~ kubectl get ns
~ helm repo list
~ helm repo update
~ helm install my-prometheus prometheus-community/prometheus \
   -n monitoring \
   --create-namespace \
   --set server.persistentVolume.enabled=false \
   --set alertmanager.persistentVolume.enabled=false

*We require above configuration for Prometheus to avoid errors

~ helm install my-grafana grafana-community/grafana --version 11.3.0 --namespace monitoring
~ kubectl get all -n monitoring
~ kubectl patch svc my-grafana -n monitoring -p '{"spec":{"type": "LoadBalancer"}}' or,
   We can also use the command “kubectl edit svc my-grafana -n monitoring” to edit in the visual editor(vi) mode
~ kubectl get all -n monitoring

*Now,Copy the Load Balancer URL

*Also get the password for admin using the command below~
	kubectl get secret -n monitoring my-grafana -o jsonpath="{.data.admin-password}" | base64 -d ; echo

~ Now Open the Load Balancer URL and login using the credentials
~ We’ll be on the Home Screen of Grafana Server
~ On the home page itself we have an option of “Data Source”
~ The datasource we’re using for Grafana is Prometheus

#Setting-Up Datasource:
~ Click on “datasource” option
~ We’ll have multiple datasource options
~ Click on Prometheus
~ Next up under Connection,Add server URL as~
	http://prometheus-server.monitoring.svc.cluster.local
~ Scroll down and click “Save and Test”
~ We’ll either get errors or Successful execution message

~ Next up, click on Dashboards and Click “Create Dashboard”
~ We can either “Add Visualisation” or “import”
~ Click on “import dashboard”
~ Here is the website URL to explore and choose a dashboard as per requirement —> https://grafana.com/grafana/dashboards/
~ However, we have to be specific to provide “Id” i.e. we choose a visualisation and it will have an ID
~ We have to specify this id and next up click “Load
~ Select “Prometheus” as the datasource
~ Click “Import”

~ We’ll now get info about different parameters
~ The Id’s are specific to monitor different parameters 
Eg.
	14623 : to monitor k8s overview in general
	15661: K8S Dashboard
	6417 : Kubernetes Cluster
	747 : pod metrics
	1860 : Node Exporter Full

~ We might get “No Data” as we have not done any deployments
~ We might get some values which are for default resources

*We can also create our own dashboard from scratch as per our requirements

*Grafana can be set for anything (including Amazon Ec2 and so on) while we are using Grafana only for K8S cluster
*We get a beautiful overview visual representation of our cluster where we can monitor everything
